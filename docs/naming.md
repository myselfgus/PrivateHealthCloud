# Naming

## Purpose

Keep product language clear, objective, and free from older terminology.

## What This Layer Does

This document defines preferred vocabulary for HealthOS Private Cloud. The repository name is PrivateHealthCloud. Voither PrivateHealthCloud can be used as a company or commercial namespace, while this scaffold uses HealthOS Private Cloud as the platform name.

## What This Layer Does Not Do

It does not create branding claims, product readiness claims, or philosophical terminology.

## Expected Interfaces

Preferred language:

- HealthOS Private Cloud
- private healthcare cloud
- Apple Silicon infrastructure
- Apple Silicon compute substrate
- Infrastructure Layer
- Data Layer
- Governance Layer
- Agent Layer
- Model Layer
- Application Layer
- Integration Layer
- governed capability
- governed projection
- safe reference
- audit event
- provenance record

Language to avoid:

- sovereign node
- single node
- local node as a product metaphor
- Core Law
- GOS
- Mestral Core
- Juridical Engine
- JAE
- constitutional tier
- tiers constitucionais

## Risks To Avoid

- Making infrastructure sound like a single machine product.
- Importing old terminology into new docs.
- Using ornamental language where contracts need precision.

## Next Steps

- Review docs and examples for forbidden terms before each release.
- Keep new service names literal and responsibility-based.
